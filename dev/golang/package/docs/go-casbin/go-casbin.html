    Golang casbin权限管理

    尹欢一的技术博客 2021/12/1

    # 一：概念：查看文档 (opens new window)

    # (一)：概念

    • casbin是一个高效的权限管理库，支持常用的多种访问控制模型，如ACL/RBAC/ABAC等，casbin具备多语言的支持

    • casbin的权限管理，其实就是控制谁能对什么资源进行什么操作，casbin将访问控制模型抽象到一个基于 PERM（Policy，Effect，Request，Matchers） 元模型的模型文件中，可以通过对模型文件的修改实现对权限的控制。模型文件中四要素概念如下

    1：request是对访问请求的抽象，它与e.Enforce()函数的参数是一一对应的

2：policy定义了具体的规则

3：matcher匹配器会将请求与定义的每个policy一一匹配，生成多个匹配结果

4：effect根据对请求运用匹配器得出的所有结果进行汇总，来决定该请求是允许还是拒绝

    # (三)：ACL权限控制

    • 创建模型文件model.conf，request_definition中的sub可以看做是subject实体，obj可以看做是object对象，act可以看做是action行为。matchers中指定了当request_definition中的sub，obj，act与policy_definition中的sub，obj，act 一一对应的时候，或者request_definition中的sub为root的时候，条件匹配，那么就放行
    [request_definition]
r = sub, obj, act

[policy_definition]
p = sub, obj, act

[matchers]
m = r.sub == p.sub && r.obj == p.obj && r.act == p.act || r.sub == "root"

[policy_effect]
e = some(where (p.eft == allow))

    package main

import (
	"fmt"
	"github.com/casbin/casbin/v2"
	"log"
)

func check(e *casbin.Enforcer, sub, obj, act string) {
	ok, _ := e.Enforce(sub, obj, act)
	if ok {
		fmt.Printf("%s CAN %s %s\n", sub, act, obj)
	} else {
		fmt.Printf("%s CANNOT %s %s\n", sub, act, obj)
	}
}

func main() {
	// 加载模型文件和策略文件，获取enforce对象
	e, err := casbin.NewEnforcer("./test/model.conf", "./test/policy.csv")
	if err != nil {
		log.Fatalf("NewEnforecer failed:%v\n", err)
	}


	check(e, "robby1", "goods_data", "read")
	check(e, "robby1", "goods_data", "write")
	check(e, "robby2", "goods_data", "read")
	check(e, "robby2", "goods_data", "write")
	check(e, "root", "all_data", "read_write")
}

// 输出结果如下
robby1 CAN read goods_data
robby1 CANNOT write goods_data
robby2 CANNOT read goods_data
robby2 CAN write goods_data
root CAN read_write all_data

    # 二：RBAC权限控制

    # (一)：概念

    • RBAC 权限模型通过引入角色 Role 这个中间层来解决用户与权限之间的问题，例如用户A是Developer角色，Developer角色对goods_data有 read 权限，那么用户A就拥有goods_data的 read 权限

    # (二)：使用RBAC设置用户权限

    • 创建模型文件model.conf，role_definition中指定了用户与角色的关系，因此判断用户是否具有权限，只需要判断r.sub是否是p.sub角色即可
    [request_definition]
r = sub, obj, act

[policy_definition]
p = sub, obj, act

# 定义角色模型
[role_definition]
g = _, _

# 只需要满足r.sub属于p.sub角色，就是满足权限
[matchers]
m = g(r.sub, p.sub) && r.obj == p.obj && r.act == p.act

[policy_effect]
e = some(where (p.eft == allow))

    • casbin支持用户和资源都具有角色，例如用户A属于Developer角色，资源Goods_data属于Dev角色，如果Developer角色对Dev有 read 权限，那么用户A对Goods_data就具有 read 权限

    # (四)：多层角色

    • casbin还能为角色定义所属角色，从而实现多层角色关系。例如用户A属于SRE角色，SRE角色属于Developer角色，那么用户A就具备Developer角色的所有权限，因此在定义权限的时候，可以给Dev角色设置共有的权限，然后额外为SRE角色定义一些特殊的权限

    # (五)：RBAC domain

    • 在casbin中，角色可以是全局的，也可以是特定domain的。例如 用户A属于Developer角色，Developer角色在 domain1 中对goods_data有 read和write权限，但Developer角色在 domain2 中对goods_data只有read权限，那么用户A在 domain1 中对goods_data有 read和write权限，在 domain2 中对goods_data只有read权限

    • 创建模型model.conf文件，在request_definition和policy_definition添加了一个dom

    [request_definition]
r = sub, dom, obj, act

[policy_definition]
p = sub, dom, obj, act

[role_definition]
g = _, _, _

[matchers]
m = g(r.sub, p.sub, r.dom) && r.dom == p.dom && r.obj == p.obj && r.act == p.act

[policy_effect]
e = some(where (p.eft == allow))

    # 三：ABAC权限控制

    # (一)：概念

    • ABAC可以实现动态属性的权限控制，例如在不同的时间段对数据data实现不同的权限控制。正常工作时间9:00-18:00所有人都可以读写data，其他时间只有数据所有者能读写。这种需求我们可以很方便地使用ABAC模型完成

    package main

import (
	"fmt"
	"github.com/casbin/casbin/v2"
	"log"
)

type Object struct {
	Name  string
	Owner string
}

type Subject struct {
	Name string
	Hour int
}

func check(e *casbin.Enforcer, sub Subject, obj Object, act string) {
	ok, _ := e.Enforce(sub, obj, act)
	if ok {
		fmt.Printf("%s CAN %s %s at %d:00\n", sub.Name, act, obj.Name, sub.Hour)
	} else {
		fmt.Printf("%s CANNOT %s %s at %d:00\n", sub.Name, act, obj.Name, sub.Hour)
	}
}

func main() {
	e, err := casbin.NewEnforcer("./test/model.conf")
	if err != nil {
		log.Fatalf("NewEnforecer failed:%v\n", err)
	}

	// 指定data的所有者是robby
	o := Object{"data", "robby"}

	// 设置robby1访问资源的时间是10点
	s1 := Subject{"robby1", 10}
	check(e, s1, o, "read")

	s2 := Subject{"robby2", 10}
	check(e, s2, o, "read")

	s3 := Subject{"robby", 20}
	check(e, s3, o, "read")

	s4 := Subject{"robby1", 20}
	check(e, s4, o, "read")
}

import (
	"fmt"
	sqlxadapter "github.com/Blank-Xu/sqlx-adapter"

	"log"
	"time"
)


var Db *sqlx.DB

func Init() {

	dsn := fmt.Sprintf("%s:%s@tcp(%s:%d)/%s?charset=utf8mb4&parseTime=True",
		"root",
		"yhy3426356",
		"127.0.0.1",
		3306,
		"sso",
	)

	defer func() {
		r := recover()
		if r != nil {
			log.Printf("database connect failed: %s\n", r)
			return
		}
	}()

	// 如果连接不上，这里会panic
	Db = sqlx.MustConnect("mysql", dsn)
	Db.SetMaxOpenConns(20)
	Db.SetMaxIdleConns(10)
	Db.SetConnMaxLifetime(time.Minute * 10)
	return

}

func check(e *casbin.Enforcer, sub, obj, act string) {

		fmt.Printf("%s CAN %s %s\n", sub, act, obj)
	} else {
		fmt.Printf("%s CANNOT %s %s\n", sub, act, obj)
	}
}

func main() {
	// 初始化数据库连接
	Init()

	if err != nil {
		panic(err)
	}

		panic(err)

	check(e, "robby2", "goods_data", "read")
