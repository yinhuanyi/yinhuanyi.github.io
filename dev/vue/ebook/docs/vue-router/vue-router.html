    # （三）路由重定向和嵌套路由

    • 当访问/goods的时候，将路由重定向到/goods/vegetable，且Vegetable组件和Fruit组件是局部组件。修改index.js中路由配置如下。访问/goods的时候，路由重定向到/goods/vegetable，那么在App.vue根组件中，<router-view />会先显示Goods组件，在Goods组件的<router-view />中，会显示Vegetable组件。如果访问/goods/fruit，在App.vue根组件中，<router-view />会先显示Goods组件，在Goods组件的<router-view />中，会显示Fruit组件。但是如果Goods组件中没有<router-view />，那么将不会显示Vegetable和Fruit组件了。可以看出嵌套路由可以设计嵌套组件

        redirect: '/goods/vegetable', // 访问/goods的时候，路由重定向到/goods/vegetable
        children: [
            {
                path: '/goods/vegetable',
                component: Vegetable,
            },
            {
                path: '/goods/fruit',
                component: Fruit,
            }
        ],
    },
    {
        path: '/user',
        component: User
    },

    # （一）Vue-Router的实现原理

    • vue-router 实例化时会初始化 this.history，不同 mode 对应不同的 history
    constructor (options: RouterOptions = {}) {
    this.mode = mode
    
    switch (mode) {
      case 'history':
        this.history = new HTML5History(this, options.base)
        break
      case 'hash':
        this.history = new HashHistory(this, options.base, this.fallback)
        break
      case 'abstract':
        this.history = new AbstractHistory(this, options.base)
        break
      default:
        if (process.env.NODE_ENV !== 'production') {
          assert(false, `invalid mode: ${mode}`)
        }
    }
}

    # 三：Promise对象和Async、Await修饰符

    # （一）Promise对象

    • Promise是一种异步非阻塞的事件驱动模型解决方案，一般用于执行一些一异步操作，例如网络请求等。某个线程执行Promise的时候，此时可以让出CPU，当请求返回的时候，会基于事件驱动的方式，通知当前线程继续执行。Promise有三种状态如下。Promise对象的状态改变有两种：从pending到resolved、从pending到rejected
    1：pending：初始化状态

2：fulfilled(resolved)：操作成功状态

3：rejected：操作失败状态
    1
    2
    3
    4
    5
    • 为什么使用Promise
    1：避免回调地狱：一个函数的输出是另外一个函数的输入，是为了解决异步操作函数里的嵌套问题，只能在回调里处理异常

2：支持并发请求：Promise可以支持多个并发的请求，获取并发请求的数据

3：提高代码可读性：异步代码嵌套，可读性差

4：解决信任问题：Promise结束状态无法改变，then中的回调只会被调用一次

    # （三）Await修饰符

    • await修饰符比较特别，它必须写在async修饰的函数里面，因此有的时候，看到一个函数被async修饰，那么基本上可以发现，函数内部有await修饰符。await修饰符修饰一个函数调用，这个函数调用返回的结果是promise的resolve参数，并且当代码运行到await的时候，线程会阻塞，一遍让出CPU，当异步任务结束后，获取到数据，抢夺CPU，让线程向下执行，这就是为什么需要使用await修饰符的原因
    async function add() {

  // 获取resolve的参数值
  const result = await new Promise((resolve, reject)=>{

    setTimeout(function(){
      resolve({ name: "robby", age: 30 })
    }, 3000)

  })
  
  console.log(result)
  console.log("hello")
  
}
