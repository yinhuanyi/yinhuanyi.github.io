    # （三）Mutation功能

    • mutation用于变更Store中的数据
    1：只能通过mutation变更Store数据，不可用直接操作Store中的数据

2：mutation操作Store数据能够被集中式监听到
    1
    2
    3
    • (第一种触发方式)mutation操作Store数据count，让其加一个step步长值
    1：定义mutation
const store = new Vuex.Store({
	state: {
		count: 0
	},
	mutations: {
		// step是传递的参数
		add: function (state, step) {
			state.count += step
		}
	}
})

2：触发mutation
methods: {
	handler1: function () {
		this.$store.commit('add', 3)
	}
}

    # （四）Actions功能

    • Action用于处理异步任务
    如果通过异步操作变更数据，必须通过Action，而不能使用Mutation，但是在Action中还是需要触发Mutation的方式间接变更数据
    1
    • Store中定义的actions代码如下
    const store = new Vuex.Store({
	
	mutations: {
		add(state, step) {
			state.count += step
		}
	},
	actions: {
		addAsync(context, step) {
			setTimeout(()=>{
				// 这里还是调用了mutations中的方法
				context.commit('add', step)
			}, 1000)
		}
	}
})

    const store = new Vuex.Store({
	
	mutations: {
		add(state, step) {
			state.count += step
		}
	},
	actions: {
		addAsync({ commit, state, dispatch }, step) {
			// 代码省略
			......
		}
	}
})

    # （五）Getter功能

    • Getter用于对Store中的数据进行加工处理，从而形成新的数据，类似计算属性的功能
    1：Getter可以对Store中已有的数据加工处理之后，形成新的数据，类似Vue的计算属性

2：Store中数据发生变化，Getter的数据也会发生变化
    1
    2
    3
    • 下面是getter在Store中的定义
    const store = new Vuex.Store({
	
	state: {
		count: 0
	},
	
	getters: {
		showNum: function (state) {
			return state.count + 1
		}
	}
	
})
