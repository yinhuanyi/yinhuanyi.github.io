    # （二）Composition API

    • 回顾vue2中的Options API
    Options API：当组件越来越复杂的时候，组件可读性越来越差。因为在data中定义的变量，可以在任何方法中被使用到，使得开发过程变得困难
    1
    • Composition API
    Composition API：当组件在data中定义变量后，可以将对这个变量所有的方法都放在一起，使得数据定义和数据使用在一起，让开发过程聚焦，提高开发效率
    1

    # （三）使用了 Proxy 代替 Object.defineProperty() 实现响应式

    • Object.defineProperty()是通过为对象属性指定描述符的方式来监听对象中某个属性的 get 和 set

    • Proxy用于为对象创建一个代理，从而实现基本操作的拦截。 那么这样就从根本上解决了 Object.defineProperty() 所面临的那么两个问题。这也是为什么 vue3 中不再有 Vue.set 方法的原因

    # （四）全新的全家桶

    • vue 被称为是 渐进式框架，就是因为，对于 vue 而言，它不仅仅只有 vue.js 这一个核心库，还有其他的比如 vue-router，vuex等一些周边库，这些周边库和 vue.js 一起共同组成了 vue

    # （五）全新的typescript支持

    • vue 3 使用 TypeScript 进行了重构，其目的是 为了防止随着应用的增长，而产生的许多潜在的运行时静态类型的错误。同时这也意味着以后在 vue 中使用 TypeScript 不再需要其他的任何工具

    • Vite可以替代使用webpack打包

    # （二）登录页面的图标处理方案

    • 一般项目中所使用的 icon 图标，一共分为两类
    1：element-plus 的图标

2：当前项目自定义的 svg 图标组件
    1
    2
    3
    • 那么对于自定义图标组件，它就需要拥有两种能力
    1：显示外部 svg 图标

2：显示项目内的 svg 图标
    1
    2
    3
    • 那么在项目开发过程中，如果使用element-plus图标就会很简单，直接应用就好了。如果使用自定义的svg图标，就需要开发一个SvgIcon图标组件。在components下创建SvgIcon/index.vue组件，在SvgIcon组件的视图模板中，有两个根组件，用于显示外部svg和内部svg，根据父组件传递的icon属性来判断是内部还是外部，如果icon属性是以https、mailto、tel开头，那么就是外部svg，否则就是内部svg。父组件还可以传递className属性，对子组件的样式修改。定义了三个计算属性，用于协议icon属性的变化
    <!-- 对图标组件的封装 -->
<template>
  <!-- 展示外部图标 -->
  <div
    v-if="isExternal"
    :style="styleExternalIcon"
    class="svg-external-icon svg-icon"
    :class="className"
  ></div>

  <!-- 展示内部图标 -->
  <svg v-else class="svg-icon" :class="className" aria-hidden="true">
    <use :xlink:href="iconName" />
  </svg>
</template>

<script setup>
// 这里是定义子组件的props
import { defineProps, computed } from 'vue'
// 这里给函数取了一个别名
import { isExternal as external } from '@/utils/validate'

// 接收父组件的props属性
const props = defineProps({
  // 图标名称
  icon: {
    type: String,
    required: true
  },
  // 图标类名称
  className: {
    type: String,
    default: ''
  }
})

// 判断当前图标是否是外部图标
const isExternal = computed(() => external(props.icon))

// 外部图标样式
const styleExternalIcon = computed(() => ({
  mask: `url(${props.icon}) no-repeat 50% 50%`,
  '-webkit-mask': `url(${props.icon}) no-repeat 50% 50%`
}))

// 内部图标：给所有的图标添加前缀
const iconName = computed(() => `#icon-${props.icon}`)
</script>

export const setItem = (key, value) => {
  // 将数组、对象类型的数据转化为 JSON 字符串进行存储
  if (typeof value === 'object') {
    value = JSON.stringify(value)
  }
  window.localStorage.setItem(key, value)
}

// 获取数据
export const getItem = (key) => {
  const data = window.localStorage.getItem(key)
  try {
    return JSON.parse(data)
  } catch (err) {
    return data
  }
}

// 删除数据
export const removeItem = (key) => {
  window.localStorage.removeItem(key)
}

// 删除所有数据
export const removeAllItem = (key) => {
  window.localStorage.clear()
}

          // Todo: 跳转到 /路由，但是不应该在这里跳转，应该在login组件中跳转
          router.push('/')
          resolve()
        })
        .catch((error) => {
          // 抛出异常
          reject(error)
        })
    })
  }

    # （六）登录鉴权解决方案

    • 什么是登录鉴权
    1：当用户未登陆时，不允许进入除 login 之外的其他页面。

2：用户登录后，token 未过期之前，不允许进入 login 页面

    • 创建permission.js文件，在路由的前置守卫中，对每次路由跳转进行拦截
    import router from './router'
import store from './store'

// 白名单
const whiteList = ['/login']
/**
 * 路由前置守卫
 * to: 要到哪里去
 * from：你从哪里来
 * next：是否要去
 */
router.beforeEach(async (to, from, next) => {
  // 存在 token ，进入主页
  // 快捷访问
  console.log('beforeEach')
  console.log(store.getters.token)
  if (store.getters.token) {
    if (to.path === '/login') {
      next('/') // 如果用户访问的是/login，那么跳转到首页
    } else {
      next() // 如果访问的不是/login，那么直接跳转即可
    }
  } else {
    // 不存在token，如果访问白名单路由，那么直接跳转即可
    if (whiteList.indexOf(to.path) > -1) {
      next()
    } else {
      next('/login') // 如果访问的不是白名单，强制跳转到/login
    }
  }
})
