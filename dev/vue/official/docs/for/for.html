    # 一：Vue中v-for指令

    # （一）用 v-for指令 把一个数组对应为一组元素

    • 我们可以用 v-for 指令基于一个数组来渲染一个列表。v-for 指令需要使用 item in items 形式的特殊语法，其中 items 是源数据数组，而 item 则是被迭代的数组元素的别名
    <body>

    <div id="app">
        <li v-for="item in items" :key="item.message">
            {{ item.message }}
        </li>
    </div>


    <script type="text/javascript">

        var app = new Vue({
            el: '#app',
            data: {
                items: [
                    {message: 'Foo'},
                    {message: 'Bar'}
                ]
            },
        })

    </script>

</body>

    # （三）维护状态

    • 当 Vue 正在更新使用 v-for 渲染的元素列表时，它默认使用“就地更新”的策略。如果数据项的顺序被改变，Vue 将不会移动 DOM 元素来匹配数据项的顺序，而是就地更新每个元素，并且确保它们在每个索引位置正确渲染。这个默认的模式是高效的，但是只适用于不依赖子组件状态或临时 DOM 状态 的列表渲染输出

    • 为了让Vue跟踪到每个节点，需要为每项提供一个key 属性，这就是为什么在Vue开发过程中，经常使用遍历索引index作为key属性的值

    <div v-for="(item, index) in items" :key="index">
  <!-- 内容 -->
</div>

    # 二：Vue中对数组的操作

    # （一）数组更新

    • 在Vue中，数组有内建的方法，当调用这些方法的时候，数组会发生改变，进而触发视图更新
    push()：向数组添加元素到末尾

pop()：从数组中删除最后一个元素

shift()：从数组中删除第一个元素

unshift()：向数组添加元素到开头

splice()：将多个元素添加到数组某个位置

sort()：对数组进行正向排序

reverse()：对数组进行反向排序

    # （三）显示数组过滤或排序的结果

    • 有时候我们想要显示一个数组经过过滤或排序后的结果，而不实际变更原始数据，那么就可以使用计算属性来解决
    <body>

    <div id="app">
        <ul>
            <li v-for="(n, index) in evenNumbers" :key="index"> {{ n }} </li>
        </ul>
    </div>


    <script type="text/javascript">

        var app = new Vue({
            el: '#app',
            data: {
                numbers: [1,2,3,4,5]
            },
            computed: {
                evenNumbers: function () {
                    // 只保留偶数元素
                    return this.numbers.filter(function (number) {
                        return number % 2 === 0
                    })
                }
            },

        })

    </script>

</body>

    # 三：v-for指令的使用范围

    # （一）v-for可以作用在标签上，遍历多少次就会渲染出多少个标签

    • 作用在普通标签<span>上
    <div>
  <span v-for="n in 10">{{ n }} </span>
</div>
    1
    2
    3

    # （二）在 template 上使用 v-for

    • 类似于v-if，v-for可以作用在template标签，来循环渲染一组元素
    <body>

    <div id="app">
        <ul>
            <template v-for="(item, index) in items" :key="index">
                <li>{{ item.msg }}</li>
                <span class="divider" role="presentation">{{ index }}</span>
            </template>
        </ul>
    </div>


    <script type="text/javascript">

        var app = new Vue({
            el: '#app',
            data: {
                items: [
                    { msg: '花丛' },
                    { msg: '绿草' },
                    { msg: '建筑' }
                ]
            }

        })

    </script>

</body>

    # （三）v-for 与 v-if 一同使用

    • 不推荐在同一元素上使用 v-if 和 v-for，当v-if 和 v-for处于同一节点，v-for 的优先级比 v-if 更高，当你只想为部分项渲染节点时，这种优先级的机制会十分有用，下面代码只渲染未完成的 todo
    <li v-for="todo in todos" v-if="!todo.isComplete">
  {{ todo }}
</li>
    1
    2
    3
    • 目的是有条件地跳过循环的执行，那么可以将 v-if 置于外层元素
    <ul v-if="todos.length">
  <li v-for="todo in todos">
    {{ todo }}
  </li>
</ul>

<p v-else>No todos left!</p>

    # （四）组件上使用v-for

    • 在自定义组件上，你可以像在任何普通元素上一样使用 v-for
    <my-component v-for="(item, index)in items" :key="index"></my-component>
    1
    • 任何数据都不会被自动传递到组件里，因为组件有自己独立的作用域，组件需要声明数据的接收。例如下面的my-component组件，items是父组件的数据，从父组件获取item后，基于v-bind:item="item"将数据传递到子组件，子组件必须声明一个props属性，用来接收item的数据，以便子组件可以显示item的数据。那么数据从父组件传递到子组件再显示的过程为：Vue根实例挂载到某个节点，父节点的数据传递到视图，视图中在子组件标签里遍历父节点数据，从而将数据传递到子组件，在子组件通过props属性接收数据，然后显示到template模板上
    <body>

    <div id="app">
        <ul>
            <my-component
                    v-for="(item, index) in items"
                    v-bind:item="item"
                    v-bind:index="index"
                    :key="index"
            ></my-component>
        </ul>
    </div>

	
    <script type="text/javascript">
		
        // 定义todo-item子组件
        Vue.component('my-component', {
            template: '<li>{{ index }} - {{ item.msg }} </li>',
            props: ['item', 'index']
        })
		
		// 1：Vue实例通过
        var app = new Vue({
            el: '#app',
            data: {
                items: [
                    {msg: 'hello'},
                    {msg: '你好'},
                    {msg: '哈哈哈'}
                ]
            },

        })

    </script>

</body>
