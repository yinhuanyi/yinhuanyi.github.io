    尹欢一的技术博客 2021/3/1
    • 对Vue.js的页面渲染原理的理解
    1：Vue脚手架在渲染的过程中，首先访问的是main.js文件，在main.js文件中会配置axios请求的默认配置，包括baseURL、timeout、interceptors，且将axios绑定到Vue框架中，那么在其他组件中使用axios的时候，直接this.axios.get()使用axios即可。最后实例化Vue这个类，将App组件挂载到public/index.html页面中带id=app的div上面。

2：App.vue组件中，template部分会有一个预留<router-view></router-view>标签，router-view是展示所有页面的入口，前端请求经过router.js文件，渲染的组件会在这里展示。

3：在router.js文件中定义了路由的匹配规则，当前端访问/的时候，会渲染pages/home.vue组件，将home.vue组件渲染后挂载到App.Vue组件的router-view标签中进行展示。在home.vue组件中也有一个router-view标签，是用来展示router.js中定义的children组件的。

4：局部组件的命名一般是驼峰式的命名，局部组件一般会挂载到页面组件中，例如NavFooter.vue会挂载到home.vue组件中，在页面组件home.vue中引用局部组件NavFooter.vue的方式很简单，只需要在script中导入局部组件NavFooter.vue，在template中写一个<nav-header></nav-header>标签即可。
