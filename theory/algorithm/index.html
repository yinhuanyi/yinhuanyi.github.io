    # 一：算法与数据结构基础

    # (一)：线性查找算法

    • 1：简要介绍了什么是算法、算法的时间复杂度

    # (二)：排序基础

    • 1：选择排序思想：外层循环只管向后遍历，内层循环向后遍历的过程中，找到最小元素的索引，然后将最小元素与外层循环所指向的索引交换位置

    • 2：插入排序思想： 外层循环只管往后遍历，内层循环向前遍历的时候，只要比前面的数小就往前插

    # (三)：动态数组、栈、队列

    • 1：动态数组本质上是一种静态的线性结构，底层是数组。向动态数组添加元素的时候，需要判断是否需要扩容，如果需要扩容会重新创建一个底层数组，将之前数组元素拷贝到新数组中。动态数组删除元素的时候，需要考虑缩容。数组删除中间元素会使得后面的元素都向前移动

    • 2：动态数组可以作为栈的底层数据结构

    • 3：动态数组可以作为队列的底层数据结构

    • 4：之所以需要循环队列，是因为队列的底层是动态数组，删除头部元素是O(n)的时间复杂度，因为在动态数组头部删除一个元素，那么其他元素都将会向前移动，因此时间复杂度是O(n)，那么循环队列正好解决了这个问题。在循环队列中，删除头部元素，只需要front指针向后移动即可，时间复杂度为O(1)

    • 5：基于数组实现的数据结构的优势在于能够快速检索，因为底层是一块连续的内存空间，但是如果大量插入或者删除元素，会使得数组元素频繁的被移动，降低性能

    # (四)：链表

    • 1：链表是一种动态的线性数据结构，有三种类型的实现：单向链表、双向链表、循环链表。

    • 2：单向链表对头部元素的CRUD操作，时间复杂度为O(1)，因此特别适合作为栈的底层数据结构。可以实现带索引的单向链表，也可以实现不带索引的单向链表

    • 3：单向链表实现队列的时候，设置可以头尾指针，让头部元素和尾部元素的CRUD操作时间复杂度为O(1)

    # 二：高级排序算法 和 二分搜索树

    # (一)：归并排序算法

    • 1：归并排序算法思想
    基于递归的思想，将一个数组从中间分为两份，不断递归，直到子数组中只有一个元素或者没有元素的时候，停止迭代。然后将两边的数组进行合并，合并的时候会用到临时拷贝的数组
    1
    • 2：归并排序的核心是Merge操作，合并子数组
    1：如果左边子数组第一个元素比mid大，说明左边子数组为空，那么右边子数组的第一个元素就是最小元素(将最小元素覆盖原数组对应的元素即可)

2：如果右边子数组第一个元素大于r，说明右边子数组为空，那么左边子数组的第一个元素就是最小元素

3：如果左边子数组的第一个元素小于右边第一个元素，那么左边子数组的第一个元素就是最小元素

4：如果右边子数组的第一个元素小于左边第一个元素，那么右边子数组的第一个元素就是最小元素

    # (二)：快速排序算法

    • 1：快速排序算法思想
    快速排序主要是先对原始数组进行partition操作，找到一个合适的索引V，让索引左边的元素小于V，索引右边的元素大于V，然后迭代再对左边的元素和右边的元素进行partition操作，直到左边或右边元素为1个或者为空，停止迭代。这样原始数组就排好序了
    1
    • 2：快速排序算法的核心是partition操作，找到合适的索引V
    假设数组的第一个元素为V，l是数组第一个元素，r是数组最后一个元素，j为【V】左边的最后一个元素，i为【V】右边的第一个元素。此时arr[l+1…j] 是空数组，arr[j+1…i-1]也是空数组，i遍历数组的时候，如果后面一个元素比V大，那么直接i++。如果后面一个元素比V小，那么让j+1, 然后将i指向的元素与j+1指向的元素交换位置，再i++，直到i>r，停止遍历

    # (三)：二分查找算法

    • 1：使用二分查找算法的前提是数组元素必须有序，二分查找算法的时间复杂度是O(logn)，因为CRUD操作都是递归进行的，递归的时间复杂度取决于树的深度

    • 2：二分查找算法思想：在有序数组中，找到中间的元素，判断【中间元素】与需要查找的元素的大小，如果【中间元素】> 【需要查找的元素】，就在左边的数组区间查找。如果【中间元素】< 【需要查找的元素】，就在右边的数组区间查找

    • 3：使用二分查找算法，在一个有序数组中，找到大于某个元素的最小值：这个问题使用二分查找算法，不断的从数组中获取【中间元素】，让【中间元素】与target比较，每次循环都改变数组的区间，直到数组的左边界对于右边界停止循环，此时如果还是没有找到对应的target，就将左边界的索引返回，因为左边界对应的值就是大于target的最小值

    • 4：在golang中，sort.Search函数就是一个二分查找算法，需要传递一个闭包函数。二分搜索算法可以应用在一致性hash算法中，寻找hash切片中，大于等于target的最小hash值，然后基于hash值从hash环上(map结构)获取对应点value值，也就是服务器的IP值，从而实现hash一致性算法

    # (四)：二分搜索树

    • 1：二分搜索树的性质：二分搜索树每个节点的值，大于其左子树所有节点的值，小于其右子树所有节点的值

    • 2：前序遍历：根节点 -> 左节点 -> 右节点；中序遍历：左节点 -> 根节点 -> 右节点；后序遍历：左节点 -> 右节点 -> 根节点

    • 3：二分搜索树的最小值：一直走到最左边走不动了就是最小值；二分搜索树的最小值：一直走到最右边走不动了就是最大值

    • 4：二分搜索树中删除节点分为三种情况

    1：删除最小值节点：一直向左走，当左节点为nil的时候，当前节点就是最小值节点，那么需要保存右边节点，让右边节点返回，覆盖当前的根节点即可

2：删除最大值节点：一直向右走，当右节点为nil的时候，当前节点就是最大值节点，那么需要保存左边节点，让左边节点返回，覆盖当前的根节点即可

3：待删除的节点既有左节点，又有右节点：使用Hibbard Deletion方法删除，首先找到以当前节点为根节点的后继节点(右子树的最小值)，然后删除这个后继节点，并且将这个后继返回，覆盖当前的根节点即可

    # 三：更多的树型结构

    # (一)：堆(Treap 树堆)和优先队列

    • 1：满二叉树的性质：除了叶子节点，其他所有的节点都具有左右两个子节点。

    • 2：完全二叉树的性质：当把元素按照一层一层顺序排列成树型结构，这样的二叉树就是完全二叉树。完全二叉树的最后一层叶子节点可能不会满，但是都会集中在最左边，倒数第二层可能会有叶子节点，但是都会集中在最右边。完全二叉树具有平衡性，但是不一定是平衡二叉树，因为平衡二叉树强调的是有序

    • 3：平衡二叉树的性质：树的左右子树的高度差不超过1的数，空树也是平衡二叉树的一种，平衡二叉树不一定是完全二叉树，因为有可能右子树深度比左子树深度多1

    • 4：二叉堆的性质：二叉堆的元素存储在动态数组中，只是添加元素的时候，不是按照动态数组的方式添加元素了，而是按照二叉堆的性质添加元素，也就是说，动态数组添加元素后按照二叉堆的性质重新排列。也就是说添加元素不再添加到动态数组的尾部，而是添加到一个指定的位置，这个指定的位置由最大二叉堆的性质决定。二叉堆删除元素只能删除堆顶元素，与添加元素同理，删除堆顶元素后也需要维护二叉堆的性质

    • 5：最大二叉堆的性质：二叉堆是一颗完全二叉树，也是平衡二叉树，堆中某个节点的值一定大于等于其子节点的值

    • 6：最小二叉堆的特征：二叉堆是一颗完全二叉树，堆中某个节点的值一定小于等于其子节点的值

    • 7：优先队列：优先队列的底层是基于二叉堆实现的，例如入队是调用二叉堆的Add方法，出队是调用二叉堆的ExtractMax方法，获取队首元素是调用二叉堆的FindMax方法

    • 8：优先队列可以方便的解决Top K问题：例如给定已知数组：{4, 5, 1, 6, 2, 7, 3, 8}，找出其中最小的4个数

    解题思路：
    先从数组中选择前4个元素，添加到优先队列中，然后其他的元素与优先队列中队首元素(最大元素)进行比较，如果比队首元素小，那么调用优先队列的出队方法，在将新的元素添加到优先队列。最后将优先队列元素输出到数组中返回
    1
    2

    • 9：Golang的container/heap包实现了二叉堆

    • 10：快速排序和优先队列 在处理Top K问题或Select K问题的比较：如果是已知全量数据的情况下，快排比优先队列更具有优势，但是如果是流式数据排序，那么使用优先队列会更好一些

    从算法时间复杂度对比

	快排：时间复杂度 O(nlogn)，空间复杂度 O(1)
	优先队列：将数组形成二叉堆时间复杂度 O(n), 空间复杂度为O(n)，替换二叉堆的堆顶元素时间复杂度是O(logn)，因此在解决Top K问题上，优先队列的时间复杂度为O(nlogn)

    # (二)：冒泡排序、希尔排序、排序算法总结

    • 1：冒泡排序算法思想：外层循环只管向后遍历，内层循环向后遍历过程中，如果当前元素比后一个元素大，那么交换位置，且内层循环每次遍历都减少一个元素

    • 2：希尔排序算法思想：希尔排序是插入排序的改进，建立在插入排序的基础上实现的，希尔排序采用的分割策略，将原有数组分割多个小组，对不同的小组再进行插入排序

    • 3：希尔排序的核心是分组思想：对每次分组进行插入排序

3：对元素间隔为n/8的所有子数组进行插入排序

3：对元素间隔为n/16的所有子数组进行插入排序

......

n：对元素间隔为1的所有子数组进行插入排序，最终让数组全部有序

    # (三)：线段树

    • 1：什么是线段树：线段树就是一段区间，对于给定的一段区间，更新区间的一个或多个元素，或者查询一个区间中，最大值，最小值，计算区间数字和

    • 2：线段树不是完全二叉树，线段树是平衡二叉树

    • 以求和业务为例，根节点是子节点的和，那么根节点的值有两个子节点组成，那么这是一个递归的结构，想要获取到根节点的值就需要获取到子节点的值

    • 3：在线段树实现中，声明了线段树的merge属性是一个函数类型，目的是让用户决定传递什么样的逻辑函数，从而实现什么样的业务逻辑，示例中是求和业务逻辑

    • 4：线段树的更新和查询操作的时间复杂度：O(logn)

    # (四)：Trie字典树

    • 1：Trie树是一个前缀树，可以实现快速插入和查询

    • 2：Trie树最大的缺点是占用空间大，因此很多项目中使用Compressed Trie进行了优化

    # (五)：AVL树

    • 1：节点高度计算：叶子节点的高度是1，非叶子节点的高度是左右节点高度的最大值再加1

    • 2：节点平衡因子计算：左节点高度-右节点高度的绝对值，那么有平衡二叉树的定义，判断一颗树是否是平衡二叉树，主要判断各个左子树和右子树的高度差不能超过1，也就是各个节点的平衡因子最大是1

    • 3：二分搜索树不一定是平衡二叉树，AVL树一定是平衡二叉树，也是二分搜索树

    • 4：AVL树是在二分搜索树的基础上，维护了树的平衡性(添加节点或者删除节点，始终让各个节点的平衡因子最大是1)。维护树的平衡性核心在于使用LL、RR、LR、RL四种旋转方式对树的结构进行调整

    • 5：AVL树的CRUD操作时间复杂度为log(n)

    # (六)：基于2-3树的红黑树

    • 1：2-3树的性质：2-3树是一颗绝对平衡树(类似满二叉树)，任意节点到达叶子节点的深度相等
    1：满足二分搜索树的基本性质

2：节点可以存放一个元素或者两个元素

3：每个节点或者有2个孩子，或者有3个孩子

    • 2：一颗2-3树可以转变为红黑树：只需要将融合节点设置为红色，那么就是一颗红黑树。因为2-3树是绝对平衡树，使得在红黑树中从任意一个节点到叶子节点，经过的黑色节点是一样的，红黑树是保持“黑平衡”的二叉树，因此n个节点的红黑树最大深度是2logn，因为在2-3树的基础上，多了红色节点。红黑树的CRUD操作时间复杂度为log(n)

    • 3：红黑树添加元素和删除元素的时候，维护黑节点平衡的核心是：使用【左旋转】、【右旋转】、【颜色翻转】三种情况的组合

    • 4：对比二分搜索树、AVL树、红黑树的性能和使用场景

    二分搜索树
	优点：对于完全随机的数据，普通的二分搜索树很好用
	缺点：极端情况下，二分搜索树会退化成为链表

AVL树
	优点：对于查询较多的情况下，AVL树性能较高
	缺点：AVL是绝对平衡树，添加和删除元素比红黑树多

红黑树
	优点：红黑树仅仅是黑平衡树，因此插入和删除节点，修复平衡能力比AVL树强
	缺点：红黑树牺牲了二叉树的平衡性(树的深度是2logn)，不满足平衡二叉树的性质

综合分析：
	红黑树的查询性能略微逊色于AVL树，因为其比AVL树会稍微不平衡最多多一层，也就是说红黑树的查询性能只比相同内容的AVL树最多多一次比较，但是，红黑树在插入和删除上优于AVL树，AVL树每次插入删除会进行大量的平衡度计算，而红黑树为了维持红黑性质所做的红黑变换和旋转的开销，相较于AVL树为了维持平衡的开销要小得多

    # (七)：哈希表

    • 1：引出哈希表：给定一个字符串，找到第一个不重复的字符，并且返回它的索引，如果不存在，返回-1
    这里的字符与数组索引的一一映射就是一个哈希表
    1
    • 2：哈希表需要解决两个问题：哈希函数 和 哈希冲突
    1：需要找到一个函数，将“键”转换为“索引”，这个找到的函数就是哈希函数

2：在哈希函数对“键”进行转换的过程中，可能不同的“键”会转换为同一个“索引”，这种情况叫哈希冲突，需要避免哈希函数的哈希冲突

    1：一致性：如果a==b, 那么 hash(a)==hash(b)

2：高效性：计算高效简便

3：均匀性：哈希值均匀分布

    • 4：链地址法：解决哈希冲突
    	当两个不同的元素，基于哈希函数计算后，得到的值相等，这种情况称之为：哈希冲突。哈希冲突是一个概率性事件，可以使用链地址法解决哈希冲突，也就是使用链表或者红黑树解决哈希冲突。

    • 7：Set和Map可以使用二分搜索树(红黑树)实现，也可以使用哈希表实现：使用二分搜索树实现的Set和Map时间复杂度为O(logn)，使用哈希表实现的Set和Map时间复杂度为O(1)。时间复杂度的差异本质在于哈希表是无序的，二分搜索树是有序的

    # (七)：SQRT分解

    • 1：SQRT分解思想的核心： SQRT分解使用分块、分组思想解决区间问题。与优先对队列一样，适合处理动态数据，算法时间复杂度为O(sqrt(n))

    • 2：SQRT分解的核心是分组：把一个含有N个元素的数组分成n½份，例如18个元素会被分为5份。修改数据的时间复杂度是O(1)，因为只需要修改区间段数据和，查询效率依然为O(sqrt(n))

    # 四：非比较排序与模式匹配

    # (一)：计数排序

    • 1：计数排序的思想：通过统计相同元素的个数，确定不同类型的元素边界，使得相同元素在一起，从而实现排序。类似问题有：给定一个包含红色、白色、蓝色，一共n个元素的数组，对这个数组进行排序，我们使用0、1、2分别表示红色、白色、蓝色。计数排序使用了计数统计的方法对元素进行规整，最终让数组有序

    # (二)：模式匹配算法

    • 1：暴力搜索解决字符串匹配问题：(【问题】：给定一个数组和一个目标子串，从数组中找出第一个与目标子串匹配的子串，并且返回其索引)字符串一个一个进行匹配，是一种暴力模式匹配算法。暴力搜索最坏的情况下，时间复杂度是O(mn)，最好的情况下时间复杂度是O(n)。因此暴力搜索的方式解决字符串匹配可能会出现算法性能退化

    • 2：解决字符串匹配的问题：哈希解决字符串匹配问题，主要是提升算法的性能，将字符串的逐一比对，转换为哈希比对

    【段式回文问题】：例如 “volvo”可以分为”vo”，”l”， “vo”，无论是正着读还是反着读，都是一样，volvo可以分为3段。那么给定的字符串，返回其最大的段

1：暴力枚举思想：设置i和j，从数组的左边界和右边界出发，不断对数组从两端向中间迭代，任何一段相等都说明有2段回文，判断相等的条件为：s[left:i+1] == s[j:right+1]


2：哈希思想：哈希思想与暴力枚举思想类似，主要是在比较字符串是否相等使用了哈希比较，那么就需要计算哈希值，如果哈希相等在比较字符串是否相等

    【问题】：给定一个字符串，找出其中的子串，即是前缀，也是后缀。例如”level”，l即是前缀也是后缀。

1：暴力枚举思想：从左右最长的子串开始遍历，最长子串的长度为字符串长度减一，然后让左右子串不断缩小，直到相等，那么此时找到的子串就是快乐前缀

2：滚动哈希思想：子串滚动的时候，先加上最后一个字符的哈希值，将子串添加到map中，再减去第一个字符的哈希值，保证每次循环子串的哈希都是9个字符的hash值。这里由于使用的是10进制数表示hash值，且只有4个不同字符的子串，计算子串哈希的过程中，不需要对素数取模，那么就不存在hash冲突问题
