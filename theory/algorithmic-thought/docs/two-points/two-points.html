    • 双指针总结：双指针处理数组问题指的是：一开始i指针指向索引0的位置，j指针指向索引为len(nums)-1的位置(一开始，i指针和j指针都指向索引为0或者1的指针不是双指针)。双指针的目的是让i和j所指向的区间不断缩小，直到i>j跳出循环。双指针处理链表问题指的是：分别指向两个链表的head节点

    # (一)：双指针

    • 可以使用双指针方法：使用i指针和j指针，一开始i指向数组第一个元素，j指向数组最后一个元素，计算当前面积，比较当前面积和maxAreaHeight大小。然后开始向中间移动指针，移动哪个指针呢？谁的数比较小，就移动哪个指针，重复此操作，让两个指针不断靠拢，直到i指针对于j指针退出循环。时间复杂度为O(n)
    func min(x,y int) int {
	if x > y {
		return y
	}
	return x
}


func maxArea(height []int) int {
	maxAreaHeight := 0

	// 枚举法
	for i, j:= 0, len(height)-1; i<j; {

		length := j - i
		innerHeigh := min(height[i], height[j])
		area := length * innerHeigh

		if area > maxAreaHeight {
			maxAreaHeight = area
		}

		// 哪个指针指向的值小就移动哪个指针
		if height[i] < height[j] {
			i++
		}else {
			j--
		}

	}

	return maxAreaHeight

}

    # (一)：第二层循环使用双指针

    • 在遍历数组的时候，先让数组从小到大排序。保证第一层遍历的指针i，所指的元素不重复。第二层遍历的指针j从i+1开始，k从最后一个元素开始，也需要保证j所指的元素不重复，当j遍历一个元素的时候，就判断是否需要向前移动k的指针，向前移动k指针的条件是nums[i] + nums[k] + nums[j]是大于0的，如果是小于0，那么k向后移动指针就好了。当j与k重合的时候，就需要跳出第二层循环了，如果nums[i] + nums[k] + nums[j]等于0，那么就是我们需要找到的值。具体代码实现如下，时间复杂度为O(n2)
    func threeSum(nums []int) [][]int {

	// 先对元素进行排序，保证数组元素是从小到大排列的
	sort.Ints(nums)

	targetNums := [][]int{}

	for i:=0; i<len(nums); i++ {

		// 保证了和上次枚举的值不同
		if i > 0 && nums[i] == nums[i-1] {
			continue
		}

		//在第二层循环中，设置了两个指针，一个是j，从前向后。一个是k，从后向前。
		//在第二层循环里面，j会先移动一个指针，由于数组是有序的，k向前移动的时候，会使得 nums[i] + nums[k] + nums[j] 减小，一旦小到不大于0就停止，那么就需要判断是否j和k是否重合了
		//如果重合了，说明j和k的值都找完了都没有找到nums[i] + nums[k] + nums[j]等于0的值，那么就跳出第二层循环，让i向后移动。
		//如果没有重合，那么就判断一下nums[i] + nums[k] + nums[j]是否等于0，如果是就添加到数组，如果不等于0 ，说明nums[i] + nums[k] + nums[j] < 0了，那么就让k向后移动
		//这样保证了i所指元素不重复，j所指元素不重复，k所指元素不重复
		for j, k:=i+1, len(nums) - 1; j<len(nums); j++ {

			// 保证了和上次枚举的值不同
			if j > i+1 && nums[j] == nums[j-1] {
				continue
			}


			// k向前移动的条件是：当k大于j，并且三个指针指向的数相加大于0
			for k > j && nums[i] + nums[k] + nums[j] > 0 {
				k--
			}

			// 如果k向前移动到了j指向的位置，那么说明重合了，那么j和k的值就找完了
			if k == j {
				break
			}

			// 如果nums[i] + nums[k] + nums[j]等于0，添加到列表中
			if nums[i] + nums[k] + nums[j] == 0 {
				targetNums = append(targetNums, []int{nums[i], nums[k], nums[j]})
			}

		}
	}

	return targetNums

}



func main() {

	ret := threeSum([]int{-1,0,1,2,-1,-4})
	fmt.Println(ret)

}

    # (一)：排序 + 第二层循环双指针

    • 算法实现如下
    func threeSumClosest(nums []int, target int) int {

	sort.Ints(nums)


	// 最好的三数和
	best := math.MaxInt32


	// cur是当前三数和，如果cur-target的距离小于best-target的距离，那么cur赋值给best
	update := func(cur int) {
		if abs(cur - target) < abs(best - target) {
			best = cur
		}
	}

	// 枚举 a
	for i := 0; i < len(nums); i++ {

		// 保证和上一次枚举的元素不相等
		if i > 0 && nums[i] == nums[i-1] {
			continue
		}
		// 使用双指针枚举 b 和 c
		j, k := i + 1, len(nums) - 1
		for j < k {
			sum := nums[i] + nums[j] + nums[k]
			// 如果和为 target 直接返回答案
			if sum == target {
				return target
			}
			// 更新最好的三数和best的值
			update(sum)

			// 如果 sum > target
			if sum > target {
				// 如果和大于 target，让k向后移动，减小当前的sum值
				k0 := k - 1

				// 移动到下一个不相等的元素
				for j < k0 && nums[k0] == nums[k] {
					k0--
				}
				k = k0
				
			} else {
				// 如果和小于 target，让j向后移动，增加当前的sum值
				j0 := j + 1
				// 移动到下一个不相等的元素
				for j0 < k && nums[j0] == nums[j] {
					j0++
				}
				j = j0
			}

		}
	}
	
	return best
}

    # 四：删除排序数组中的重复项

    给你一个 升序排列 的数组 nums ，请你 原地 删除重复出现的元素，使每个元素 只出现一次 ，返回删除后数组的新长度。元素的 相对顺序 应该保持 一致 。

由于在某些语言中不能改变数组的长度，所以必须将结果放在数组nums的第一部分。更规范地说，如果在删除重复项之后有 k 个元素，那么 nums 的前 k 个元素应该保存最终结果。

将最终结果插入 nums 的前 k 个位置后返回 k 。

不要使用额外的空间，你必须在 原地 修改输入数组 并在使用 O(1) 额外空间的条件下完成。

判题标准:

系统会用下面的代码来测试你的题解:

int[] nums = [...]; // 输入数组
int[] expectedNums = [...]; // 长度正确的期望答案

int k = removeDuplicates(nums); // 调用

assert k == expectedNums.length;
for (int i = 0; i < k; i++) {
    assert nums[i] == expectedNums[i];
}
如果所有断言都通过，那么您的题解将被 通过。

 

示例 1：
输入：nums = [1,1,2]
输出：2, nums = [1,2,_]
解释：函数应该返回新的长度 2 ，并且原数组 nums 的前两个元素被修改为 1, 2 。不需要考虑数组中超出新长度后面的元素。

示例 2：
输入：nums = [0,0,1,1,1,2,2,3,3,4]
输出：5, nums = [0,1,2,3,4]
解释：函数应该返回新的长度 5 ， 并且原数组 nums 的前五个元素被修改为 0, 1, 2, 3, 4 。不需要考虑数组中超出新长度后面的元素。

    # (一)：i指针移动，j指针删除重复元素：复杂度O(n2)

    • 算法中使用j指针，通过j指针判断是否与i指针指向的元素相等，如果相等说明需要删除j指向的元素，删除的时候需要判断一下是否j+1越界，如果越界，那么只删除j指向的元素就可以了。第二次外层循环，i会执行不同的元素，算法复杂度是O(n2)
    func removeDuplicates(nums []int) int {

	for i:=0; i<len(nums); i++ {

		j := i+1

		for j<len(nums) && nums[i] == nums[j] {
			// 更新数组
			if j+1 < len(nums) {
				nums = append(nums[0:j], nums[j+1:]...)
				// 判断i指向的数是否与j指向的数相等，由于num缩短一个，相等的情况下，再j++，那么重复的就无法删除
				if nums[j] != nums[i]{
					j++
				}

			}else {
				nums = nums[0:j]
				break
			}

		}

	}
	return len(nums)
}

func main() {
	ret := removeDuplicates([]int{0,0,0,0})
	fmt.Println(ret)
}

    # (二)：双指针：复杂度O(n)

    • 双指针：通过fast指针遍历数组，通过slow指针更新数组，当fast指针向后遍历过程中，如果判断当前fast指针所指的数与上一个数不同，让这个数赋值给slow指针指向的数，然后slow++。最后返回slow的索引即可，就是不同元素的长度。算法复杂度是O(n)，由于只是填充和统计，并未删除原始数组中的重复元素，因此效率高
    func removeDuplicates(nums []int) int {

	n := len(nums)
	if n == 0 {
		return 0
	}

	slow := 1
	
	for fast := 1; fast < n; fast++ {
		if nums[fast] != nums[fast-1] {
			nums[slow] = nums[fast]
			slow++
		}
	}

	return slow

}

func main() {
	ret := removeDuplicates([]int{0,0,1,1,1,2,2,3,3,4})
	fmt.Println(ret)
}

    # (一)：哈希表

    • 这个方法比较简单，就是将第一个链表的所有节点加入到哈希表中，第二个链表的节点加入哈希表的时候，先判断一下是否节点已经存在，如果存在就返回。算法复杂度为O(n+m)，空间复杂度为O(n+m)

    # (二)：双指针法

    • 如果pa和pb不相交，会一直运行下去，直到pa和pb为nil返回。如果pa等于pb那么直接跳出循环，返回pa即可。时间复杂度为O(n)，空间复杂度为O(1)
    func getIntersectionNode(headA, headB *ListNode) *ListNode {
    if headA == nil || headB == nil {
        return nil
    }
    pa, pb := headA, headB
    for pa != pb {
        if pa == nil {
            pa = headB
        } else {
            pa = pa.Next
        }
        if pb == nil {
            pb = headA
        } else {
            pb = pb.Next
        }
    }
    return pa
}
