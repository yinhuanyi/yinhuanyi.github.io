    # 二：Elasticsearch中名词的解释

    • Elasticsearch名词
    1：集群cluster	
   由一个或多个节点组成，对外提供服务

2：节点Node:
	一个elasticsearch节点

3：索引Index：
	由具有相同字段的文档列表组成(必须是相同字段)

4：索引类型Type：
	目前只允许有一个类型doc_

5：文档类型mapping：
	用于指定document的字段名称和类型，可以自己指定，也可以自动生成
	在7.x以后，一个mapping只属于一个index

6：文档Document：
	用户存储在ES中的数据文档，相当于数据表中的一行数据

7：分片Shard：
	将索引划分为多份的能力，运行水平划分和扩展，多个分片响应请求，提高性能和吞吐量。一个索引的数据，可以发布在多个分片上

8：副本Replicas：
	创建分片的一份或多份的能力

    • Document文档字段数据类型(重要)
    1：简单数据类型：
	字符串类型：text, keyword
	数值类型：long, integer, short byte, double, float, half_float, scaled_float
	布尔：boolean
	日期：date
	日期(纳秒)：data_nanos
	二进制：binary
	范围类型：integer_range, float_range, long_range, double_range, date_range

2：复杂数据类型
	Array: 数组类型	
	Object：单个json对象类型
	Nested：数组的json对象类型

3：地理数据类型
	Geo-point： geo_point （for lat/lon points）
	Geo-shape: geo_shape (for complex shapes like polygons)

4：特殊数据类型
	IP:  ip (IPv4 和 IPv6 地址)
	Completion类型：completion （to provide auto-complete suggestions）
	Token count：token_count (to count the number of tokens in a string)
	mapper-murmur3：murmur3(to compute hashes of values at index-time and store them in the index)
	mapper-annotated-text：annotated-text （to index text containing special markup (typically used for identifying named entities)）
	Percolator：（Accepts queries from the query-dsl）
	Join：（Defines parent/child relation for documents within the same index）
	Alias：（Defines an alias to an existing field.）
	Rank feature：（Record numeric feature to boost hits at query time.）
	Rank features：（Record numeric features to boost hits at query time.）
	Dense vector：（Record dense vectors of float values.）
	Sparse vector：（Record sparse vectors of float values.）
	Search-as-you-type：（A text-like field optimized for queries to implement as-you-type completion）

    • text与keyword的区别（最大的区别就是text不支持直接索引、排序、聚合查询）
    text类型
	1:支持分词，全文检索,支持模糊、精确查询, 不支持聚合,排序操作
    2:test类型的最大支持的字符长度无限制,适合大字段存储
	使用场景：
    存储全文搜索数据, 例如: 邮箱内容、地址、代码块、博客文章内容等。
    默认结合standard analyzer(标准解析器)对文本进行分词、倒排索引。
    默认结合标准分析器进行词命中、词频相关度打分。


keyword类型
	1:不进行分词，直接索引,支持模糊、支持精确匹配，支持聚合、排序操作。
	2:keyword类型的最大支持的长度为——32766个UTF-8类型的字符,可以通过设置ignore_above指定自持字符长度，超过给定长度后的数据将不被索引，无法通过term精确匹配检索返回结果。
	使用场景：
	存储邮箱号码、url、name、title，手机号码、主机名、状态码、邮政编码、标签、年龄、性别等数据。
	用于筛选数据(例如: select * from x where status='open')、排序、聚合(统计)。
	直接将完整的文本保存到倒排索引中。

    • document中的元数据（比较重要）
    1：_index: 文档所在的索引名称
2：_type: 文档所在的类型名称, 现在只有一个_doc类型
3：_id: 文档唯一的id
4：_version：文档的版本信息，保证了文档变更以正确顺序执行，避免了乱序造成文档丢失
5：_seq_no：严格递增的顺序号，保证了后写入的document大于先前写入的document
6：_primary_term：当主shard重新分配，或重新选举主节点，
7：found：如果文档找到了就是true
8：_source: 文档的原始json数据，可以从这里获取每个字段的内容
9：_score：理想的排序是和查询条件相关性越高排序越靠前，而这个排序的依据就是_score
