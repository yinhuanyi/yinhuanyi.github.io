    # 一：ElastAlert入门

    # (一)ElastAlert介绍

    • ElastAlert从0.2版本开始，必须使用Python3.6

    • 轻松灵活的发出Elasticsearch告警

    ElastAlert是一个简单的框架，用于从Elasticsearch中的数据中发出异常，峰值或其他感兴趣模式的警报。
ElastAlert适用于所有版本的Elasticsearch。
在Yelp，我们使用Elasticsearch，Logstash和Kibana来管理不断增长的数据和日志。 Kibana非常适合可视化和查询数据，但是我们很快意识到，它需要一个辅助工具来警告数据中的不一致情况。为此，创建了ElastAlert。
如果您有近乎实时地将数据写入Elasticsearch并希望在该数据与某些模式匹配时收到警报，则ElastAlert是为您提供的工具。如果您可以在Kibana中看到它，则ElastAlert可以对其发出警报。

    • 摘要(重写，用自己的使用心得叙述)
    我们将ElastAlert设计为可靠，高度模块化，并且易于设置和配置。
它通过将Elasticsearch与两种类型的组件（规则类型和警报）结合起来工作。定期查询Elasticsearch，并将数据传递到规则类型，该规则类型确定何时找到匹配项。发生匹配时，将为该警报提供一个或多个警报，这些警报将根据匹配采取行动。
这是由一组规则配置的，每个规则定义一个查询，一个规则类型和一组警报。
ElastAlert包含几种具有通用监视范例的规则类型：

1：频率匹配：在某段时间内，关键字出现的频率
2：频率的峰值匹配：频率增加或减少匹配
3：至少多少次匹配：在某一段时间内，少于X个事件时匹配
4：字段匹配：当某个字段与黑名单/白名单匹配时匹配
5：事件匹配：匹配与给定筛选器匹配的任何事件
6：比较匹配：当一个字段在一段时间内有两个不同的值时进行匹配
7：新内容匹配：当字段中出现从未见过的术语时匹配
8：阈值匹配：当字段的唯一值数高于或低于阈值时匹配

    • 主要告警类型
    any：只要有匹配就报警

blacklist：compare_key字段的内容匹配上blacklist数组里任意内容；

whitelist：compare_key字段的内容一个都没能匹配上whitelist数组里内容；

change：在相同query_key条件下，compare_key字段的内容，在 timeframe范围内 发送变化；

frequency：在相同query_key条件下，timeframe 范围内有num_events个被过滤出 来的异常；

spike：在相同query_key条件下，前后两个timeframe范围内数据量相差比例超过spike_height。其中可以通过spike_type设置具体涨跌方向是- up、down、both。还可以通过threshold_ref设置要求上一个周期数据量的下限，threshold_cur设置要求当前周期数据量的下限，如果数据量不到下限，也不触发；

flatline：timeframe范围内，数据量小于threshold阈值；

new_term：fields字段新出现之前terms_window_size(默认30天)范围内最多的terms_size(默认50)个结果以外的数据；

cardinality：在相同 query_key 条件下，timeframe范围内cardinality_field的值超过 max_cardinality或者低于min_cardinality

    # (三)ElastAlert配置

    • 在主配置文件的模板config.yaml.example基础上修改全局配置
    首先将主配置文件的模板config.yaml.example，拷贝一份为：config.yaml

1：rules_folder：ElastAlert将加载规则配置文件的地方，它将尝试加载文件夹中的每个.yaml文件。
2：run_every：ElastAlert查询Elasticsearch的频率。
3：buffer_time：是查询窗口的大小，从每个查询运行的时间向后延伸。对于其中use_count_query或use_terms_query设置为true的规则，此值将被忽略。
4：es_host：是Elasticsearch集群的地址，ElastAlert将存储有关其状态、查询运行、警报和错误的数据。每个规则也可以设置不同的elasticsearch主机进行查询。
5：es_port：Elasticsearch对应的端口。
6：use_ssl： （可选的）是否使用TLS;连接到es_host；设置为True或False。
7：verify_certs： （可选的）是否验证TLS证书; 设置为True或False，默认是True。
8：client_cert： （可选的）PEM证书的路径。
9：client_key： （可选的） 作为客户端密钥使用的私钥文件的路径。
10：ca_certs： （可选的） 用于验证SSL连接的CA证书的路径。
11：es_username： （可选的） 用于连接Elasticsearch的basic-auth用户名。
12：es_password： （可选的） 用于连接Elasticsearch的密码。
13：es_url_prefix： （可选的） Elasticsearch端点的URL前缀。
14：es_send_get_body_as： （可选的） 查询Elasticsearch方法- GET，POST或source，默认是GET。
15：writeback_index：是ElastAlert将存储数据的索引名称。
16：alert_time_limit： 是失败警报的重试窗口。

    • ElastAlert查询
    ElastAlert 根据elastalert_status去确定首次启动的时候在什么时间范围内去查询，以避免重复查询。对于每个规则，它将从最近的结束时间开始查询。包括：
rule_name：相应规则的名称。
starttime：查询的开始时间戳。
endtime：查询结束时间戳。
hits：查询结果的数量。
matches：处理命中后规则返回的匹配数。请注意，这并不一定意味着警报被触发。
time_taken：此查询运行所需的秒数。

    • 编写一个告警推送脚本
    #! /usr/bin/env python
# coding: utf-8

import sys
import requests
from datetime import datetime

# 获取告警级别
level = sys.argv[1]
# 获取告警@的用户
username = sys.argv[2]
# 获取告警关键字
content = sys.argv[3]
# 获取日志中主机的IP
ip = sys.argv[4]
# 获取日志中主机的主机名
hostname = sys.argv[5]
# 获取主机角色
role = sys.argv[6]
# 获取日志索引
index = sys.argv[7]
# 获取日志路径
log_path = sys.argv[8]
# 获取日志信息
message = sys.argv[9]

# 将告警内容推送到Fil-Manager
if level == 'warning':
    ret = requests.get('http://10.102.0.14:8000/monitor/log_warning/',params={'username': username, 'content': content, 'ip': ip, 'hostname': hostname, 'role': role, 'index': index, 'log_path': log_path, 'message': message})
    with open('/app/elastalert/logs/alert_warning.log', 'a') as f:
        print('{} {} {} {} {} {} {} {} {}'.format(datetime.now(), ret.text, content, ip, hostname, role, index, log_path, message), file=f)
elif level == 'disaster':
    ret = requests.get('http://10.102.0.14:8000/monitor/log_disaster/',params={'username': username, 'content': content, 'ip': ip,'hostname': hostname, 'role': role, 'index': index, 'log_path': log_path, 'message': message})
    with open('/app/elastalert/logs/alert_disaster.log', 'a') as f:
        print('{} {} {} {} {} {} {} {} {}'.format(datetime.now(), ret.text, content, ip, hostname, role, index, log_path, message), file=f)

    • 在example_rules目录下，编写告警规则
    # 规则名称, 必须唯一
name: No Space
# 指定ES地址
es_host: yw3-2
es_port: 9200
# 规则类型
type: any
# 指定只要匹配就告警
realert:
  minutes: 0
# 匹配的索引
index: storage-*
# 指定时间戳字段
timestamp_field: "@timestamp"
# 指定ES的查询规则
filter:
- query:
    match_phrase:
        "message": {"query": "no space left on device"}

# 告警方式
alert:
- "command"

# 指定执行的命令
command: [
  "/app/elastalert/alert.py",
  "warning",
  "shenmingxing",
  "no space left on device",
  "%(fields.ip)s",
  "%(host.name)s",
  "%(type)s",
  "%(_index)s",
  "%(log.file.path)s",
  "%(message)s"
]
