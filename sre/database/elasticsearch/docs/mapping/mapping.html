    # 一：Mapping的概念

    # (一) Elasticsearch中的Mapping类似于数据库中的表结构schema，用于指定索引的数据类型

    • Mapping的作用
    1：定义 Index 下的字段名称

2：定义字段类型，比如数值类型，字符串类型，布尔类型

3：定义倒排索引相关的配置，比如是否索引、记录position

    # (四)自定义Mapping

    • 自定义Mapping可以新增字段，通过dynamic参数来控制新增字段
    dynamic: true(默认)，允许自动新增字段
dynamic: false，不允许自动新增字段，但是给索引插入文档的时候，字段可以正常写入，查询这个document可以看到字段内容，但是无法被query到
dynamic: strict，不允许自动新增字段，不允许写入，写入新增字段会报错

    # (七)自定义Mapping中的index_options

    • index_options的作用
    index_options用于控制倒排索引返回的内容，有如下4种配置
	①：docs只记录doc id
	②：freqs记录doc id和term frequencies
	③：positions记录 doc id、term frequencies和term position
	④：offsets 记录doc id、term frequencies、term position和character offsets


text类型的字段默认是position，其他的默认是docs
记录内容越多，占用空间越大

    ES从2.X版本一下子跳到了5.X版本，将string类型变为了过期类型，取而代之的是text和keyword数据类型，一直到现在最新的6以上版本。接下来就看看这两个字段的区别。
    
     按照官方文档的阐述，text类型的数据被用来索引长文本，例如电子邮件主体部分或者一款产品的介绍，这些文本会被分析，在建立索引文档之前会被分词器进行分词，转化为词组。经过分词机制之后es允许检索到该文本切分而成的词语，但是text类型的数据不能用来过滤、排序和聚合等操作。
    keyword类型的数据可以满足电子邮箱地址、主机名、状态码、邮政编码和标签等数据的要求，不进行分词，常常被用来过滤、排序和聚合。
    综上，可以发现text类型在存储数据的时候会默认进行分词，并生成索引。而keyword存储数据的时候，不会分词建立索引，显然，这样划分数据更加节省内存。

    这样，我们映射了某一个字段为keyword类型之后，就不用设置任何有关分词器的事情了，该类型就是默认不分词的文本数据类型。而对于text类型，我们还可以设置它的分词类型，如下：

    # (十一) ES中的Dynamic Templates

    • 允许根据ES自动识别的数据类型、字段名称来动态设定字段类型，可以实现如下效果
    ①：所有字符串类型都设定为 keyword 类型， 让默认不分词。默认的字符串类型是text，是分词的

②：如果字段以message开头，就设定为 keyword 类型
    1
    2
    3
    • 举例说明(在dynamic_templates中，可以设置多个动态模板，并且每个模板的名称可以自己取，比如【strings_to_text】这个名称，就是我自己取的名字)
    {
    "settings":{
        "number_of_shards":50,
        "number_of_replicas":0
    },
    "mappings":{
        "properties":{
            "created":{
                "type":"date"
            },
            "message":{
                "type":"keyword"
            },
            "retweets":{
                "type":"long"
            },
            "user":{
                "type":"keyword"
            }
        },
        "dynamic_templates":[
            {
                "strings_to_text":{
                    "match_mapping_type":"string",
                    "match":"text_*",
                    "mapping":{
                        "type":"text",
                        "fields":{
                            "raw":{
                                "type":"keyword",
                                "ignore_above":256
                            }
                        }
                    }
                }
            },
            {
                "strings_to_keywords":{
                    "match_mapping_type":"string",
                    "mapping":{
                        "type":"keyword"
                    }
                }
            },
            {
                "longs_to_keywords":{
                    "match_mapping_type":"string",
                    // 表示匹配字段名称以long开头，不以text结尾
                    "match":"long_*",
                    "unmatch":"*_text",
                    "mapping":{
                        "type":"keyword"
                    }
                }
            }
        ]
    }
}

    # (十三) ES中如何使用索引模板

    • 索引模板，英文为Index Template，主要是用于新建索引时，自动使用预先定义好的索引配置，能够简化索引的创建步骤
    1：可以指定索引的settings和mapping

2：可以有多个模板，根据order设置，order大的会覆盖小的配置
    1
    2
    3
    • 创建一个自定义模板
    PUT _template/my_template
{
  "index_patterns": ["my*", "bar*"],  # 需要匹配的索引名称
  "order": 0,                         # order顺序
  "settings": {
    "number_of_shards":1,
    "number_of_replicas":1
  },
  "mappings": {
    "_source": {
      "enabled": false # 默认不记录原始的数据
    },
    "properties": {
      "name": {
        "type": "keyword"
      }
    }
  }
}
